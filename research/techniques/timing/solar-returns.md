# Solar Returns

## Technique Profile

- Practice role: supporting annual layer
- Doctrine: modern / hybrid
- Birth-time sensitivity: required
- Status: implemented

## What Question This Answers

What is the shape and emphasis of the year from birthday to birthday?

## Use When

- you want the annual overlay after identifying the broader chapter
- you need a yearly emphasis chart
- you are comparing how a year may feel in different locations

## Do Not Use When

- the birth time is weak
- you want the solar return to stand alone without natal context
- you are treating one angular return placement as a total verdict

## How to Read It

1. Read the return Ascendant and its ruler.
2. Read the return Moon and any planets on return angles.
3. Overlay the return onto the natal chart.
4. Use transits to identify when the yearly themes are triggered.

## What Not to Do

- Do not treat solar returns as a replacement for natal analysis.
- Do not over-read small details when the chart has only one or two clear annual themes.
- Do not skip location if the annual question is relocation-sensitive.

## Current App Support

`node astro.mjs solar-return`

The app computes the exact return moment and the return houses for a chosen location.
