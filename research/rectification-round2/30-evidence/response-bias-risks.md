# Response Bias Risks

Rectification interviews are exposed to all the normal survey failures plus a few astrology-specific ones.

## Main Risks

| Risk | What it looks like | Why it matters | Mitigation |
|---|---|---|---|
| Barnum effect | The user endorses broad descriptions that feel true | produces false confidence | ask for contrasts and disconfirmers |
| Self-attribution | The user reshapes memory to fit the chart | inflates apparent fit | anchor on dated records where possible |
| Hindsight fitting | After seeing a candidate chart, the user starts agreeing with it | leaks model expectations back into the answer | blind the user to candidate specifics |
| Demand characteristics | The user tries to answer the "astrology" version of themselves | creates performative responses | use neutral wording and forced-choice formats |
| Social desirability | The user answers with the socially preferred option | hides awkward truths | reduce evaluative wording, use pairwise items |
| Anchoring | Early candidate charts bias later answers | narrows the search too early | keep early interview steps chart-agnostic |
| Overfitting to one event | One dramatic memory dominates the whole session | distorts the posterior | require multiple independent examples |
| Careless responding | The user answers too quickly or mechanically | corrupts the evidence stream | response-time and consistency checks |

## Astrology-Specific Failure Modes

### 1. Chart confirmation loop
Once a candidate chart is named, people tend to reinterpret everything through it. This can make the session feel successful while actually reducing information gain.

### 2. Trait inflation
Generic self-descriptions like "creative", "sensitive", or "intense" are too easy to accept. They should not carry much weight unless they are tied to concrete behavior.

### 3. After-the-fact narrative completion
Users often build a clean story from messy lives. Rectification needs the mess, not the polished biography.

### 4. Technique favoritism
Users can overvalue whichever technique sounds strongest to them. The system should not let one favorite method dominate without cross-checks.

## Interview Design Rules

1. Ask for concrete examples before interpretations.
2. Prefer comparative questions over absolute identity claims.
3. Mix domains so the user cannot infer the chart family too easily.
4. Include a "less true" or "not sure" path.
5. Keep the model blind to responses that could anchor later questions until the first screening pass is done.

## What To Measure

- consistency across repeated items
- response time by item family
- number of contradictions
- overuse of safe or flattering options
- disagreement between absolute self-ratings and forced-choice answers
